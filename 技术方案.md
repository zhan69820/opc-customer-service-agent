# 客服 Agent 技术方案

> 制定: 技术总监
> 日期: 2026-02-26

---

## 一、项目概述

- **项目名称**: OPC 客服 Agent
- **类型**: AI 对话服务
- **目标**: 为中小企业提供低成本的 AI 客服解决方案

---

## 二、技术选型

### 核心架构
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户端      │     │  服务端      │     │  大模型     │
│ (微信/网页)  │────▶│  (Next.js)  │────▶│  (API)      │
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                    ┌──────▼──────┐
                    │  知识库      │
                    │  (FAQ)      │
                    └─────────────┘
```

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js + React | 管理后台 |
| 后端 | Next.js API | 服务端 |
| 对话模型 | OpenAI / MiniMax | AI 对话 |
| 数据库 | SQLite (MVP) | 知识库存储 |
| 部署 | Docker | 一键部署 |

---

## 三、核心功能设计

### 1. 知识库管理
- 添加/编辑/删除问答对
- 支持批量导入
- 关键词设置

### 2. 对话引擎
- **优先级**: 关键词匹配 > 知识库搜索 > 大模型问答
- 支持多轮对话
- 上下文记忆

### 3. 多平台接入
- **网页客服**: 嵌入网站的聊天窗口
- **微信**: 公众号/小程序（需认证）
- **API**: 对接第三方系统

### 4. 管理后台
- 知识库管理
- 对话记录查看
- 数据统计

## 四、数据模型

### FAQ (知识库)
```typescript
{
  id: string,
  question: string,      // 问题
  answer: string,        // 回答
  keywords: string[],    // 关键词
  category: string,      // 分类
  enabled: boolean,       // 是否启用
  createdAt: Date
}
```

### Conversation (对话记录)
```typescript
{
  id: string,
  userId: string,        // 用户ID
  messages: Message[],   // 消息历史
  source: string,        // 来源(微信/网页)
  createdAt: Date
}
```

### Message
```typescript
{
  role: 'user' | 'assistant',
  content: string,
  timestamp: Date
}
```

## 五、API 设计

### 对话接口
```
POST /api/chat
Body: { message: string, userId: string, source: 'web' | 'wechat' }
Response: { reply: string }
```

### 知识库接口
```
GET    /api/faqs          # 获取问答列表
POST   /api/faqs          # 添加问答
PUT    /api/faqs/[id]    # 更新问答
DELETE /api/faqs/[id]    # 删除问答
```

## 六、部署方案

### Docker 一键部署
```yaml
# docker-compose.yml
version: '3'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - OPENAI_API_KEY=xxx
      - DATABASE_URL=./data.db
```

### 环境变量
```
OPENAI_API_KEY=sk-xxx    # 大模型 API Key
DATABASE_URL=./data.db   # 数据库路径
PORT=3000                # 端口
```
